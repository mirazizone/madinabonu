# Система загрузки фото в админ панели

## Что было сделано

### 1. Обновлена главная страница (home.blade.php)
- Фото учителей теперь загружаются из базы данных
- Если фото выбрано в админ панели - оно отображается на фронт-енде
- Если фото не выбрано - используется дефолтное изображение

### 2. Создан StorageController
- Контроллер обслуживает файлы из `storage/app/public` директории
- Защита от directory traversal атак
- Кэширование файлов для лучшей производительности

### 3. Обновлены маршруты (routes/web.php)
- Маршрут `/storage/{path}` перенаправляет на StorageController
- Файлы обслуживаются через Laravel, что даёт большую гибкость

## Как использовать

### Загрузка фото учителя в админ панели:

1. Перейти в Админ панель → Учителя
2. Создать новго учителя или отредактировать существующего
3. Выбрать фото (JPG, PNG, GIF, максимум 2MB)
4. Сохранить

### Результат:

На главной странице вместо placeholder'а появится загруженное фото.

## Структура хранилища

Загруженные фото хранятся в:
```
storage/app/public/teachers/{имя файла}
```

И доступны через URL:
```
/storage/teachers/{имя файла}
```

## Возможные проблемы

### Если фото не загружается:
1. Проверить права доступа к папке `storage/app/public`
2. Убедиться, что сервер имеет права на запись в эту папку
3. Проверить размер файла (максимум 2MB)
4. Проверить формат файла (JPG, PNG, GIF)

### Если нужна символическая ссылка:
Запустить команду (требует прав администратора):
```bash
php artisan storage:link
```

## Дополнительные возможности

Можно добавить загрузку фото для курсов аналогичным способом, обновив модель Course и контроллер CourseController.
